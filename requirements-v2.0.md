## 1. 项目概述

### 1.1 项目背景

作为教育品牌创始人，需要开发一个微信小程序，用于学员作业练习、打卡管理、积分系统和朋友圈营销宣传，提升教学管理效率和品牌影响力。

### 1.2 项目目标

- 为学员提供便捷的作业练习和打卡工具
- 建立完善的积分激励体系
- 通过朋友圈分享扩大品牌影响力
- 提供完整的教学管理后台

### 1.3 目标用户

| 用户角色 | 职责 |
|---------|------|
| 学员（家长） | 使用家长手机号注册，完成作业、打卡、获取积分 |
| 教师 | 布置作业、批改作业、管理积分、发布课堂展示 |
| 管理员 | 系统管理、权限控制、数据统计 |

### 1.4 品牌信息

| 项目 | 内容 |
|------|------|
| 品牌名称 | 百里口才\|百里演说 |
| 品牌口号 | 让演说成为孩子自信生长的力量！ |

### 3.4 关键设计原则

1. **模块化设计**：功能模块化，便于维护和扩展
2. **RESTful API**：规范的API设计
3. **安全优先**：数据加密、权限控制
4. **性能优化**：缓存策略、异步处理
5. **可扩展性**：预留扩展接口
6. **事件委托**：统一使用 `data-*` 属性 + 事件委托处理按钮点击

## 4. 数据模型设计

### 4.1 用户模型（User）

```javascript
{
  _id: ObjectId,           // 用户唯一标识
  openid: String,          // 微信openid
  phone: String,           // 手机号
  name: String,            // 用户姓名
  gender: String,          // 性别：male/female/other
  birthday: Date,          // 出生日期
  avatar: String,          // 头像Base64或URL
  role: String,            // 角色：student/teacher/admin
  points: Number,          // 当前积分
  classIds: [ObjectId],    // 所属班级ID列表
  createdAt: Date,         // 创建时间
  updatedAt: Date          // 更新时间
}
```

### 4.2 课程模型（Course）

```javascript
{
  _id: ObjectId,
  name: String,            // 课程名称
  description: String,     // 课程描述
  level: String,          // 课程级别
  courseStatus: String,   // 状态：enrolling/ongoing/ended
  suitableGrade: [String], // 适合年级
  courseIntro: String,    // 课程介绍
  courseType: String,     // 类型：main/elective/other
  courseTypeText: String, // 类型文本
  teacherIds: [ObjectId],  // 教师列表
  classIds: [ObjectId],   // 班级列表
  createdAt: Date,
  updatedAt: Date
}
```

### 4.3 班级模型（Class）

```javascript
{
  _id: ObjectId,
  courseId: { _id: ObjectId, name: String },
  name: String,            // 班级名称
  teacherIds: [{ _id: ObjectId, name: String }],
  headTeacherId: { _id: ObjectId, name: String },
  studentIds: [{ _id: ObjectId, name: String }],
  studentCount: Number,
  classType: String,       // formal/temporary
  classTypeText: String,
  startDate: Date,
  endDate: Date,
  schedule: [{ dayOfWeek: Number, startTime: String, endTime: String }],
  classroom: String,
  classStatus: String,
  classStatusText: String,
  createdAt: Date,
  updatedAt: Date
}
```

### 4.4 作业模型（Homework）

```javascript
{
  _id: ObjectId,
  courseId: { _id: ObjectId, name: String },
  classId: { _id: ObjectId, name: String },
  title: String,           // 作业标题
  content: String,         // 作业内容
  publishTime: Date,       // 发布时间
  deadline: Date,          // 截止时间
  points: Number,          // 完成积分
  cycle: {
    type: String,          // single/multi
    durationDays: Number   // 多周期时每个周期天数
  },
  frequency: {
    timesPerCycle: Number  // 每周期要求完成次数
  },
  hasCheckin: Boolean,     // 是否有打卡属性
  checkinPoints: Number,   // 打卡分享积分
  shareText: String,       // 分享文案
  createdAt: Date
}
```

### 4.5 作业提交模型（HomeworkSubmission）

```javascript
{
  _id: ObjectId,
  homeworkId: ObjectId,
  studentId: ObjectId,
  cycleIndex: Number,      // 周期索引（从0开始）
  cycleCompletionCount: Number,
  files: [{
    type: String,         // audio/video
    url: String,
    duration: Number
  }],
  submitDate: Date,
  isCompleted: Boolean,
  isShared: Boolean,
  grade: String,           // A/B/C/D
  comment: String,
  voiceComment: String,
  bonusPoints: Number,
  gradedAt: Date,
  gradedBy: ObjectId,
  createdAt: Date

```

### 4.6 打卡记录模型（Checkin）

```javascript
{
  _id: ObjectId,
  homeworkId: ObjectId,
  studentId: ObjectId,
  submissionId: ObjectId,
  date: Date,
  cycleIndex: Number,
  streak: Number,          // 连续打卡天数

```

### 4.7 积分记录模型（PointsRecord）

```javascript
{
  _id: ObjectId,
  studentId: ObjectId,
  type: String,            // earn/deduct
  source: String,         // homework/checkin/share/manual
  sourceDetail: {
    homeworkId: ObjectId,
```

## 6. 功能需求详情

### 6.1 用户认证模块

#### 6.1.1 手机号快速登录

**流程**：
1. 用户输入手机号和姓名
2. 点击"快速登录"按钮
3. 系统验证并生成 Token
4. 跳转至首页

**界面元素**：
- 手机号输入框（带+86前缀）
- 姓名输入框
- 登录按钮

#### 6.1.2 个人信息管理

**可编辑字段**：
| 字段 | 类型 | 说明 |
|------|------|------|
| 头像 | 图片上传 | 支持拍照/相册 |
| 姓名 | 文本 | 2-10个字符 |
| 性别 | 单选 | 男/女/其他 |
| 生日 | 日期选择 | 限制选择过去日期 |

### 6.2 作业模块

#### 6.2.1 作业类型

| 维度 | 类型 | 说明 |
|------|------|------|
| 周期类型 | 单次作业 (single) | 无周期划分，一次性完成 |
| | 周期作业 (multi) | 按天数划分周期，每周期需完成多次 |
| 打卡属性 | 普通作业 | 无打卡属性 |
| | 打卡作业 | 有打卡属性，计入打卡统计 |

#### 6.2.2 周期与频次计算

```
总天数 = (截止日期 - 发布日期) + 1
周期数 = ceil(总天数 / 每周期天数)
总任务数 = 每周期次数 × 周期数
```

#### 6.2.3 作业列表页

**筛选功能**：
| 筛选项 | 类型 | 选项 |
|--------|------|------|
| 课程筛选 | 下拉选择 | 全部课程 / 各课程名 |
| 状态筛选 | 下拉选择 | 全部状态 / 待完成 / 已完成 |
| 日期筛选 | 日期选择 | 筛选发布日期在该日期的作业 |

**分组展示**：
| 分组 | 标题 | 背景色 |
|------|------|--------|
| 待完成 | 📋 待完成 (n) | #FAFAFA |
| 已完成 | ✅ 已完成 (n) | #F6FFED |

### 6.3 打卡模块

#### 6.3.1 打卡统计概览

| 指标 | 说明 |
|------|------|
| 累计打卡 | 历史总打卡次数 |
| 连续打卡 | 当前连续打卡天数 |
| 最长连续 | 历史最长连续记录 |
| 今日打卡 | 当天打卡次数 |

#### 6.3.2 打卡中心入口

- 首页快捷操作 - 去打卡
- 底部导航 - 打卡

#### 6.3.3 打卡作业列表

- 仅显示 `hasCheckin=true` 的作业
- 卡片样式同作业模块

#### 6.3.4 打卡操作

**立即打卡**：
1. 自动定位到最早未完成的周期和次数
2. 自动展开周期列表和次数列表
3. 高亮显示目标任务项
4. 滚动到可见位置

**补打卡**：
- 仅在截止日期前可用
- 超过截止日期显示提示："作业已截止，无法补打卡！"

| 已完成 | 作业全部完成，可以分享！ |

#### 6.4.2 打卡作业分享

- 微信聊天：不限次数，可重复分享
- 朋友圈：每个周期限分享一次

## 8. 前端开发要求

### 8.1 代码规范

#### 8.1.1 HTML 结构

- 使用语义化标签
- 列表使用 `<ul>/<li>` 或 `<dl>/<dd>`
- 按钮统一使用 `<button>` 标签
- 图片使用 `<img>` 并设置 alt 属性

#### 8.1.2 CSS 规范

- 使用 CSS 变量管理主题色
- 使用 Flexbox 和 Grid 布局
- 避免使用行内样式（除动态样式外）
- 移动端优先设计

#### 8.1.3 JavaScript 规范

- 使用 ES6+ 语法
- 避免使用 `var`，使用 `const/let`
- 使用箭头函数
- 使用模板字符串
- 模块化组织代码

### 8.2 按钮事件处理（重要）

**统一使用 `data-*` 属性 + 事件委托**

```javascript
// HTML 模板
<button data-action="complete" data-id="${homeworkId}">完成作业</button>
<button data-action="cycle-toggle" data-id="${homeworkId}">▼</button>
<button data-action="share" data-id="${homeworkId}">分享</button>
<button data-action="checkin" data-id="${homeworkId}">立即打卡</button>
<button data-action="makeup-checkin" data-id="${homeworkId}">补打卡</button>

// JS 事件处理
document.addEventListener('click', function(e) {
  const btn = e.target.closest('[data-action]');
  if (!btn) return;
  
  const action = btn.dataset.action;
  const id = btn.dataset.id;
  
  switch(action) {
    case 'complete':
      handleHomeworkAction(id);
      break;
    case 'cycle-toggle':
      toggleCycles(id);
      break;
    case 'share':
      openShareModal(id);
      break;
    case 'checkin':
      handleCheckin(id);
      break;
    case 'makeup-checkin':
      handleMakeupCheckin(id);
      break;
    // ... 其他 action
  }
});
```

**禁止使用内联 onclick**

```javascript
// ❌ 错误
<button onclick="handleClick()">按钮</button>

// ✅ 正确
<button data-action="handle-click">按钮</button>
```

### 8.3 数据加载与缓存

```javascript
// 30秒前端缓存
const CACHE_DURATION = 30000;
let dataCache = null;
let cacheTime = 0;

async function loadData() {
  if (dataCache && (Date.now() - cacheTime) < CACHE_DURATION) {
    return dataCache;
  }
  
  const data = await fetch('/api/batch/init');
  dataCache = data;
  cacheTime = Date.now();
  return data;
}
```
